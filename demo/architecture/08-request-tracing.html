Concurrent Requests

100

Per-Request Pipeline

  1. JWT validation (~0.1ms)
  2. Scope resolution (DB query)
  3. Rule evaluation (3 rules, in-memory)
  4. Audit write (tokio::spawn, async)
  5. Response serialization (serde)

Metrics Collected

  • Wall-clock time (all 100)
  • Throughput (req/sec)
  • Per-request latency (min/max/avg)
  • Success rate (HTTP 200)
  • DB verification (SELECT COUNT)

Why Rust

Type safetyNewtype IDs at compile time
Query safetysqlx::query! checks SQL at build
PerformanceZero-cost abstractions, no GC
ConcurrencyTokio work-stealing runtime
