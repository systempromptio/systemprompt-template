Demo 01 Happy Path

Admin-scope agent receives a message, calls an MCP tool through the governance layer, and returns a structured artifact. Cost: ~$0.01. Every step produces typed IDs that link the full trace.

1 Create Context ~2ms
$ core contexts create --name "Demo 1 - Happy Path"
What happens Creates an isolated execution context. All subsequent messages, tool calls, and artifacts are scoped to this context.
Output ContextId ctx_a1b2c3d4
Rust type ContextId(String) — newtype, cannot be confused with other IDs
Why Context isolation — every agent conversation gets its own ContextId. This prevents cross-contamination between sessions and enables per-context artifact retrieval, cost tracking, and trace linking. Without it, concurrent agent sessions would share state.
2 Message Agent ~3.2s
$ admin agents message developer_agent -m "List all agents" --context-id $CONTEXT_ID --blocking --timeout 60
Agent developer_agent — scope: admin — MCP: systemprompt skill-manager
IDs created SessionId TraceId UserId (from JWT)
Internal flow
2aAI Request #1 — model receives message + tool list, decides to call mcp__systemprompt__list_agents
2bPreToolUse Hook — POST /api/public/hooks/govern → scope=admin → ALLOW ~12ms
2cMCP Execution — systemprompt server runs admin agents list → JSON result ~180ms
2dAI Request #2 — model processes tool output, formats response, creates artifact
2ePostToolUse Hook — POST /api/public/hooks/track → records tool_name, timing, bytes ~8ms
Timing AI: ~2.8s  •  Governance: ~12ms  •  MCP: ~180ms  •  Tracking: ~8ms
Trace events ~11 events — 3 AI requests, 1 MCP call, governance decisions, execution steps
Why Governance is synchronous — the PreToolUse hook blocks tool execution until the backend returns allow/deny. This is the enforcement point. If governance were async, tools could execute before the decision arrives. The PostToolUse hook is fire-and-forget (tracking only).
Why Typed IDs everywhere — UserId, SessionId, TraceId are newtypes. The compiler prevents passing a SessionId where a UserId is expected. This eliminates an entire class of audit-linking bugs and means every trace event is guaranteed to carry the correct IDs.
3 Retrieve Artifact ~1ms
$ core artifacts show $ARTIFACT_ID --full
What happens Retrieves the typed artifact the agent produced. Artifacts are structured JSON data that any surface can render — web dashboard, mobile app, Slack bot, or CLI.
Output JSON Structured agent list with names, scopes, MCP server mappings
Linked by ContextId — same context from step 1 links message → trace → artifact
Why Surface-agnostic artifacts — the agent produces typed JSON, not formatted text. Any surface (web dashboard, CLI, mobile app, Slack bot) can render the same artifact differently. This is why the artifact is stored with ContextId, not embedded in the chat response.
