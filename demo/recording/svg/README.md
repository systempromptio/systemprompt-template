# Terminal SVG recordings

These are the razor-sharp terminal SVGs embedded in the project `README.md`. They are generated by running the `svg-NN-*.sh` demo scripts under [asciinema](https://asciinema.org/) and rendering each cast with [svg-term-cli](https://github.com/marionebl/svg-term-cli).

## Layout

```
svg/
├── record.sh              # Orchestrator — regenerates all recordings
├── _colors.sh             # Shared helpers (header, type_cmd, pass/fail, …)
├── profiles/
│   ├── dark.xrdb          # Dark palette (README dark mode)
│   └── light.xrdb         # Light palette (README light mode)
├── svg-01-governance.sh   # Individual demo scripts
├── svg-02-secrets.sh      # (add svg-NN-name.sh — it's picked up automatically)
├── …
├── recordings/            # asciinema .cast files (regenerated, committed)
└── output/
    ├── dark/*.svg         # Dark-mode SVGs embedded in README
    └── light/*.svg        # Light-mode SVGs embedded in README
```

## Regenerate

```bash
just start                                         # server must be up
./demo/00-preflight.sh                             # seed + write demo/.token
just record-svgs                                   # regenerate all
just record-svgs 3 7                               # regenerate svg-03 + svg-07 only
```

`just record-svgs` shells out to `demo/recording/svg/record.sh`, which for every `svg-NN-*.sh`:

1. Runs the script under `asciinema rec` → `recordings/NN-*.cast`
2. Renders a dark SVG → `output/dark/NN-*.svg`
3. Renders a light SVG → `output/light/NN-*.svg`

## Prerequisites

| Tool | Install |
|---|---|
| `asciinema` | `pipx install asciinema` or `brew install asciinema` |
| `svg-term` | `npm i -g svg-term-cli` |
| Live server | `just start` (port 8080) |
| Auth token | `./demo/00-preflight.sh` (writes `demo/.token`) |

## Add a new recording

1. Copy an existing script, e.g. `cp svg-01-governance.sh svg-11-your-topic.sh`.
2. Source `_colors.sh`; use `header`, `subheader`, `type_cmd`, `pass`/`fail`, `divider`. Keep it under ~40 seconds of real-time playback so the SVG stays embed-friendly.
3. Hit real endpoints — no mocks. Each recording doubles as an integration smoke test.
4. Run `just record-svgs 11` to capture only the new one.
5. Commit the `svg-NN-*.sh`, the `recordings/NN-*.cast`, and the two `output/**/NN-*.svg` files together. Reference the SVG from `README.md` with a `<picture>` block so it respects dark/light mode.

## Conventions

- **Width:** 100 cols, **Height:** 32 rows. Anything wider wraps in the README column.
- **Colors:** only the palette defined in `_colors.sh`. svg-term renders anything else as default text.
- **Commands via `type_cmd`** (not raw `echo`) — the keystroke animation is what makes the recording feel alive.
- **Real data only.** Every cURL call, every CLI command runs for real against the local server. When demos drift, these scripts break loudly and visibly.
