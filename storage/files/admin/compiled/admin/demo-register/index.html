Demo Environment — User Provisioning

This page allows administrators to create user accounts and generate one-time registration links for this demo environment. Recipients click the link to register a passkey on their device, which becomes their login credential.

In production, user provisioning would be handled automatically through your organisation's identity provider (IdP) via SAML 2.0, OpenID Connect, or SCIM directory sync. This page would not exist.

How Authentication Works

This platform uses WebAuthn passkeys for passwordless authentication, the same standard used by Apple, Google, and Microsoft for phishing-resistant sign-in.

Registration

When a user clicks their registration link, their browser generates a public/private key pair bound to this domain. The private key stays on the device (or in a password manager). Only the public key is stored server-side. No shared secrets, no passwords.

Login

The server sends a random challenge. The user's device signs it with the private key. The server verifies the signature against the stored public key. This is wrapped in an OAuth 2.0 PKCE flow to issue a short-lived JWT session token.

Recovery

If a user loses their device, an administrator can generate a new registration link from this page. Magic links (time-limited, single-use tokens) bootstrap the passkey creation process on a new device.

Enterprise IdP Integration

In production, this manual provisioning is replaced by automated identity federation. Here is how each integration model works:

SAML 2.0 / OpenID Connect

Your IdP (Okta, Azure AD, Google Workspace) authenticates users. The platform receives a signed assertion with the user's identity and roles. A local session is issued without the user ever entering credentials on this platform.

SCIM Directory Sync

Users and groups are automatically provisioned and deprovisioned from your directory. When someone joins your organisation, they appear here. When they leave, access is revoked immediately.

Just-in-Time (JIT) Provisioning

On first SSO login, a user account is created automatically with roles derived from IdP group memberships. No pre-provisioning required.

Passkeys + SSO

WebAuthn passkeys can complement SSO as a device-binding factor, adding phishing-resistant MFA on top of your IdP authentication.
