# Deploy the gateway on Dokploy

Deploys the `systemprompt-gateway` server on [Dokploy](https://dokploy.com), a self-hostable PaaS. The blueprint provisions the gateway + Postgres from the GHCR image.

## Install (template)

Once the upstream template is merged, pick **systemprompt** from Dokploy's template catalog. Until then, import it manually:

1. Dashboard → your project → **Create Service** → **Template** → **Import** and paste the base64 of [`deploy/dokploy/`](https://github.com/systempromptio/systemprompt-template/tree/main/deploy/dokploy) (`docker-compose.yml` + `template.toml`), or create a **Docker Compose** service and paste `deploy/dokploy/docker-compose.yml` directly.
2. Fill in the environment variables before the first deploy:
   - `ANTHROPIC_API_KEY` and/or `OPENAI_API_KEY` / `GEMINI_API_KEY` — at least one; the container refuses to boot without a provider key. Blank values are safely ignored.
   - `POSTGRES_PASSWORD` — auto-generated by the template. If you set it manually, do so **before the first deploy** (Postgres initialises its volume with the first password it sees).
3. Deploy. First boot runs migrations and the publish pipeline — allow several minutes before `/api/v1/health` reports healthy.

## Domain + TLS

The template maps your chosen domain to the gateway's port 8080 through Dokploy's Traefik proxy, with automatic Let's Encrypt. The domain is injected as `EXTERNAL_URL`, so the gateway advertises the right public URL and CORS origin out of the box.
