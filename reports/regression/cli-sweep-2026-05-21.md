## MCP findings — `plugins mcp` and tool execution

**Tool round-trip works.** Eight `plugins mcp call systemprompt systemprompt --args '{"command":"..."}'` invocations across every domain returned `success: true` with `artifact_type: table|list|text`, execution time 120–250 ms (log 224–231). Artifact rendering survives the core 0.11 bump.

**Negative paths return proper errors:**
- Malformed JSON in `--args` → `Invalid JSON in --args` (log 233)
- Bad server name → `MCP server '<name>' not found in configuration` (log 237)
- Missing tool → `tool name required in non-interactive mode` (log 236)
- Empty args `{}` → tool executes, MCP server returns its own error about missing `command` field (log 234). Acceptable.

**Real defect — `plugins mcp list-packages`:**
```
Error: Failed to list MCP packages
Caused by:
  0: Failed to bootstrap AppContext
  1: Config not initialized. Call Config::init() first.
```
Bootstrap order regression. Every other `plugins mcp *` subcommand initializes config correctly; only `list-packages` skips it. Log 127.

**Real defect — `plugins show <id>`:**
`plugins list --json` reports 14 extensions including `systemprompt`, `web`, etc. with `id` fields. Passing any of those ids to `plugins show <id>` returns `Extension '<id>' not found`. The lookup keys diverged from the discovery keys. Log 118 and follow-up check.

**Friction — embedded MCP version drift:** The MCP server's `InitializeResult.server_info.version` reports `"0.9.2"` (log 224), but the CLI is `0.11.0` and core is `0.11`. The hard-coded version string in `extensions/mcp/systemprompt/` was not bumped during the release.

**Error paths:**
- Nonexistent agent → `Agent request failed with status 404 Not Found` (log 240)
- Missing `--message` in non-interactive mode → clear error (log 241)
- 1-second timeout → `operation timed out` on the HTTP client (log 242)

**Audit spine after A2A traffic** (logs 247–248):
- `infra logs request list` shows 10 Gemini calls with token counts, latency, $0.0002–0.0006 per call — works.
- `infra logs trace list` shows 10 traces — **but every trace has `status: "unknown"` and `ai_requests: 0`**. The `mcp_calls` count is correct (1 per trace), so the trace table is being written, but:
  1. The `status` column is never resolved from "unknown" to `ok`/`denied`/`error`.
  2. The trace is not joined to the `ai_requests` rows that we know exist for the same time window.

This is a P1 audit-correctness regression — the governance spine still records events but loses outcome and AI-cost linkage. Likely caused by the audit-schema change in `extensions/web/schema/11_audit_event_notify.sql`.

## Friction items

| # | Item | Severity | Detail |
| --- | --- | --- | --- |
| F-1 | Cloud-token-expired WARN prints on every CLI invocation | P2 | Contaminates stdout for downstream parsers; mixes with `--json` output unless explicitly redirected. The advisory log line should be gated by `--verbose` or only printed when a `cloud *` command is actually invoked. |
| F-2 | Verb naming inconsistent across domains | P2 | Several domains use unique singletons where one would expect `list`: `admin session show`, `admin config paths show`, `web sitemap show`, `analytics sessions stats`, `infra db migrations status`. Each was a wrong guess in round 1. Consider aliasing `list` → the canonical verb to reduce surprise, or standardising. |
| F-3 | MCP `serve_inner` and rmcp transport INFO logs print to stdout during `plugins mcp call` | P2 | Mixes log lines with the JSON tool result. Should be suppressed unless `--verbose`. Log 224. |
| F-4 | `plugins mcp call` JSON argument flag is non-obvious | P2 | The positional shape is `<SERVER> <TOOL>` and JSON args go via `-a/--args '<json>'`. The `--help` is correct, but the natural "just pass it as the third positional" guess fails noisily (round 1 logs 129–139). Consider accepting trailing JSON as a third positional or adding it to `--help` examples. |
| F-5 | MCP server reports stale `version` 0.9.2 | P2 | See above. Bump in `extensions/mcp/systemprompt/Cargo.toml` (or wherever the constant lives) and wire through to `Implementation { version: ... }`. |
| F-6 | `admin agents message` without quotes around multi-word `-m` value can collide with later flags | P3 | Today it works because `--blocking`/`--timeout` shift parsing, but a message ending in `--something` would break silently. Document quoting requirement in `--help`. |

**P2 — friction reduction:**
5. Gate the cloud-token-expired WARN to cloud commands only.
