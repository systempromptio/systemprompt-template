# PostgreSQL Container Management - Core Repo

Work required in `systemprompt-core` repository.

## Summary

When user creates a local tenant, automatically create and start PostgreSQL container with the specified credentials.

## Problem

Current `tenant create --local`:
1. Prompts for PostgreSQL credentials
2. Assumes PostgreSQL already running
3. Saves database_url to tenant
4. Connection fails if DB doesn't exist

## Solution

New `tenant create --local`:
1. Prompts for credentials (user, password, db name, port)
2. Generates docker-compose.yml
3. Starts PostgreSQL container
4. Waits for healthy
5. Saves tenant with working database_url

```rust
async fn create_local_tenant() -> Result<StoredTenant> {
    CliService::section("Create Local PostgreSQL Tenant");

    // 1. Collect credentials
    let name: String = Input::with_theme(&ColorfulTheme::default())
        .with_prompt("Tenant name")
        .default("local".to_string())
        .interact_text()?;

    let db_user: String = Input::with_theme(&ColorfulTheme::default())
        .with_prompt("Database user")
        .default("systemprompt".to_string())
        .interact_text()?;

    let db_password: String = Password::with_theme(&ColorfulTheme::default())
        .with_prompt("Database password")
        .interact()?;

    if db_password.is_empty() {
        bail!("Password cannot be empty");
    }

    let db_name: String = Input::with_theme(&ColorfulTheme::default())
        .with_prompt("Database name")
        .default("systemprompt".to_string())
        .interact_text()?;

    let port: u16 = Input::with_theme(&ColorfulTheme::default())
        .with_prompt("Port")
        .default(5432_u16)
        .interact_text()?;

    // 2. Check if port in use, offer to reset
    if is_port_in_use(port).await {
        let reset = Confirm::with_theme(&ColorfulTheme::default())
            .with_prompt(format!("Port {} is in use. Stop existing container and reset?", port))
            .default(false)
            .interact()?;

        if reset {
            stop_container_on_port(port).await?;
        } else {
            bail!("Port {} is in use. Choose a different port.", port);
        }
    }

    // 3. Generate docker-compose.yml
    let compose_content = generate_postgres_compose(&name, &db_user, &db_password, &db_name, port);
    let docker_dir = PathBuf::from(".systemprompt/docker");
    fs::create_dir_all(&docker_dir)?;
    let compose_path = docker_dir.join(format!("{}.yml", name));
    fs::write(&compose_path, &compose_content)?;
    CliService::success(&format!("Created: {}", compose_path.display()));

    // 4. Start container
    CliService::info("Starting PostgreSQL container...");
    let status = Command::new("docker")
        .args(["compose", "-f", compose_path.to_str().unwrap(), "up", "-d"])
        .status()?;

    if !status.success() {
        bail!("Failed to start PostgreSQL container. Is Docker running?");
    }

    // 5. Wait for healthy
    let spinner = CliService::spinner("Waiting for PostgreSQL to be ready...");
    wait_for_postgres_healthy(&compose_path, 60).await?;
    spinner.finish_and_clear();
    CliService::success("PostgreSQL is ready");

    // 6. Create tenant
    let database_url = format!(
        "postgres://{}:{}@localhost:{}/{}",
        db_user, db_password, port, db_name
    );

    let id = format!("local_{}", nanoid());
    Ok(StoredTenant::new_local(id, name, database_url))
}
```

**Add helper functions:**

```rust
fn generate_postgres_compose(
    tenant_name: &str,
    user: &str,
    password: &str,
    db: &str,
    port: u16,
) -> String {
    format!(
        r#"# SystemPrompt PostgreSQL - Tenant: {tenant_name}
# Generated by: systemprompt cloud tenant create
# Manage: just db-up {tenant_name} / just db-down {tenant_name}

services:
  postgres:
    image: postgres:17-alpine
    container_name: systemprompt-postgres-{tenant_name}
    restart: unless-stopped
    environment:
      POSTGRES_USER: {user}
      POSTGRES_PASSWORD: {password}
      POSTGRES_DB: {db}
    ports:
      - "{port}:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U {user} -d {db}"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
    name: systemprompt-postgres-{tenant_name}-data
"#,
        tenant_name = tenant_name,
        user = user,
        password = password,
        db = db,
        port = port
    )
}

async fn is_port_in_use(port: u16) -> bool {
    TcpListener::bind(format!("127.0.0.1:{}", port)).is_err()
}

async fn stop_container_on_port(port: u16) -> Result<()> {
    // Find container using the port
    let output = Command::new("docker")
        .args(["ps", "--filter", &format!("publish={}", port), "-q"])
        .output()?;

    let container_id = String::from_utf8_lossy(&output.stdout).trim().to_string();

    if !container_id.is_empty() {
        CliService::info(&format!("Stopping container {}...", container_id));
        Command::new("docker")
            .args(["stop", &container_id])
            .status()?;
        Command::new("docker")
            .args(["rm", &container_id])
            .status()?;
    }

    Ok(())
}

async fn wait_for_postgres_healthy(compose_path: &Path, timeout_secs: u64) -> Result<()> {
    let start = std::time::Instant::now();

    loop {
        let output = Command::new("docker")
            .args([
                "compose",
                "-f",
                compose_path.to_str().unwrap(),
                "ps",
                "--format",
                "{{.Health}}",
            ])
            .output()?;

        let health = String::from_utf8_lossy(&output.stdout).trim().to_string();

        if health.contains("healthy") {
            return Ok(());
        }

        if start.elapsed().as_secs() > timeout_secs {
            bail!(
                "Timeout waiting for PostgreSQL to become healthy. \
                Check logs with: docker compose -f {} logs",
                compose_path.display()
            );
        }

        tokio::time::sleep(Duration::from_secs(2)).await;
    }
}
```

## New User Flow

```bash
# 1. Create local tenant - CREATES & STARTS PostgreSQL
just tenant-create
# Select "Local"
# Enter: user, password, database, port
# → docker-compose generated at .systemprompt/docker/local.yml
# → Container started and healthy
# → Tenant saved with database_url

# 2. Create profile - only config files
just profile-create local
# → Uses tenant's database_url
# → Creates profile.yml + secrets.json only
# → Connection works because container already running

# 3. Migrate and run
just migrate
just start
```
