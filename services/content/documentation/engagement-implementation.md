## How Phase 1 Works

Phase 1 is a collaborative development phase. Both sides work together to define the requirements, build the Control Center, and evaluate the result. No production commitment is required — Phase 2 only begins after both sides sign off on Phase 1 delivery.

### Process

| Step | What Happens |
|------|-------------|
| **1. PRD Agreement** | A detailed Product Requirements Document is agreed between both sides. This defines the scope, deliverables, acceptance criteria, and timeline. |
| **2. Development** | The Control Center is built on systemprompt cloud infrastructure. Foodles has hands-on access throughout development. |
| **3. Evaluation** | Both sides work with the running system. Iterative feedback, refinements, and adjustments based on real usage. |
| **4. Sign-Off** | Formal sign-off by both parties confirms the PRD has been delivered. This is the gate to Phase 2. |

### Where It Runs

Phase 1 runs entirely on **systemprompt cloud infrastructure**. Foodles does not need to provision any servers, databases, or infrastructure during this phase. The platform is fully operational on our cloud — Foodles evaluates a working system, not a mockup.

## What We Build

The Control Center is a branded dashboard built on the platform's extension architecture. It is a standalone codebase — separate from the core platform — configured for Foodles's operations.

### Deliverables

| Deliverable | Description |
|-------------|-------------|
| **Branded dashboard** | White-label Control Center with Foodles branding, colours, and assets |
| **Skill migration** | All existing skills mapped, migrated, and configured for Foodles's operational domains |
| **Agent configuration** | Agents configured with appropriate RBAC, skills, and MCP server access |
| **Branding assets** | Logo integration, colour scheme, typography, and favicon configuration |
| **Deployment package** | Ready to deploy into Foodles's enterprise infrastructure (used in Phase 2) |
| **Documentation** | Architecture overview, configuration guide, and operational runbook |

### What Control Center Includes

The Control Center provides Foodles with a governed administration interface for managing AI agents across the organisation:

- **Agent management** — configure, monitor, and govern all agents from a single dashboard
- **Skill management** — create, edit, and assign skills to agents by department
- **User and role management** — RBAC configuration, department assignment, access control
- **Analytics dashboards** — usage, cost, and adoption metrics by department and agent
- **MCP server management** — register, configure, and monitor MCP server connections
- **Audit and compliance** — searchable audit trails, conversation logs, and compliance reporting

## Ownership

> **Foodles owns the Control Center codebase from the moment of delivery.**

The extension codebase — the dashboard, all configurations, all skills, all agents, all customisations — is Foodles's intellectual property from the moment of delivery. This is true regardless of whether Phase 2 proceeds.

This means:

- **Full source code** — Foodles receives the complete, unobfuscated source code
- **Modification rights** — Foodles can modify, extend, or fork the codebase without restriction
- **No dependency on Phase 2** — the Control Center code belongs to Foodles even if the production license is never signed
- **Internal development** — Foodles's engineering team can develop the codebase independently
- **Additional consulting** — Foodles can engage us for additional development on the Control Center at any time

The Control Center is built on the platform's extension architecture, but it is a separate codebase delivered in a dedicated repository with full Git history.

## What Happens After Phase 1

| Outcome | What Happens Next |
|---------|------------------|
| **Both sides sign off** | Phase 2 begins — platform license signed, deployed to Foodles infrastructure, goes live |
| **Foodles needs changes** | PRD is revised, additional work is scoped and quoted |
| **Foodles does not proceed** | Foodles keeps the Control Center codebase. No further obligation. |
